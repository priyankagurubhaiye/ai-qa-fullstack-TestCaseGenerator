# 🤖 AI-Powered QA Test Case Generator (Full-Stack)

## 🚀 Overview

This project is a **full-stack AI-driven QA application** that automates the process of generating test cases from software requirements.

Users can upload a requirement file, and the system uses **GenAI (LLM)** to analyze the input and generate:

* ✅ Positive test cases
* ❌ Negative test cases
* ⚠️ Edge cases

The goal is to **reduce manual QA effort**, improve test coverage, and accelerate software release cycles using an **AI-first testing approach**.

# 🤖 AI-Powered QA Test Case Generator (Full-Stack)

## 🚀 Overview

This project is a **full-stack AI-driven QA application** that automates the process of generating test cases from software requirements.

Users can upload a requirement file, and the system uses **GenAI (LLM)** to analyze the input and generate:

* ✅ Positive test cases
* ❌ Negative test cases
* ⚠️ Edge cases

The goal is to **reduce manual QA effort**, improve test coverage, and accelerate software release cycles using an **AI-first testing approach**.

---

## 🧠 What This Project Does

### 🔹 Step-by-Step Flow

1. User uploads a requirement file (e.g., login feature)
2. Backend reads and processes the requirement
3. AI model analyzes the requirement
4. Generates structured test cases
5. Displays results instantly on the UI

---

## 🏗️ Architecture

Frontend (HTML + JS)
⬇
Flask Backend (API Layer)
⬇
GenAI Engine (OpenAI API)
⬇
Test Case Output

---

## 🛠️ Tech Stack

* **Frontend:** HTML, JavaScript
* **Backend:** Python (Flask)
* **AI Engine:** OpenAI API (GenAI)
* **Other Tools:** REST API, JSON

---

## ▶️ How to Run the Project

### 1. Clone or Extract the Project

```bash
git clone https://github.com/yourusername/ai-qa-fullstack.git
cd ai-qa-fullstack
```

---

### 2. Install Dependencies

```bash
pip install -r requirements.txt
```

---

### 3. Set OpenAI API Key

#### For Windows:

```bash
set OPENAI_API_KEY=your_api_key_here
```

#### For Mac/Linux:

```bash
export OPENAI_API_KEY=your_api_key_here
```

---

### 4. Run the Application

```bash
python app.py
```

---

### 5. Open in Browser

Go to:

```
http://127.0.0.1:5000
```

---

## 📂 How to Use

1. Click on **"Choose File"**
2. Upload a `.txt` file with requirement (e.g., login functionality)
3. Click **"Generate Test Cases"**
4. View generated output instantly

---

## 📊 Example Input

```
User should be able to login using valid email and password.
Invalid login should show error message.
```

---

## 📋 Example Output

* Test Case 1: Valid login with correct credentials
* Test Case 2: Invalid login with wrong password
* Test Case 3: Empty email field
* Test Case 4: Special characters in password
* Edge Cases: Large input values, SQL injection attempts
